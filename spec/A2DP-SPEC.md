# A2DP — Agent-to-Agent Decentralized Protocol v1.0

> 对标比特币的去中心化理念，应用于 Agent 能力验证与信任建立。

---

## 一、核心问题

当前 AI Agent 生态的信任模型是**中心化的**：

```
Agent A 声称"我能做投资分析" → 谁信？
  → 因为是 OpenAI 认证的？
  → 因为跑在 GPT-5 上？
  → 因为有 VC 背书？
```

A2DP 的回答：**信任不由机构背书产生，由可验证的推理历史自证。**

## 三、Proof of Reasoning (PoR)

### 3.1 流程

```
1. Agent 收到查询 → 产出可验证预测
   例: "上证指数 Q3 突破 4100，置信度 0.75"
       验证条件: "上证收盘价 > 4100 至少 3 个交易日"
       验证窗口: "2026-10-01"

2. 预测写入账本 → 不可篡改(链式哈希)

3. 验证窗口到期 → 自动检查实际结果
   上证 Q3 最高 4050，未突破 4100 → 验证失败

4. 验证结果追加到账本(不覆写原记录)

5. 信任分重新计算 → accuracy 下降
```

### 3.2 与 PoW 的差异

| | PoW (比特币) | PoR (A2DP) |
|---|---|---|
| 证明什么 | 你消耗了算力 | 你做出了可验证的正确预测 |
| 难度 | 哈希前导零 | 预测准确率 |
| 作假成本 | 51% 算力 | 高免疫等级 Agent 的交叉验证 |
| 奖励 | 比特币 | 信任分 → 更多查询路由 |

## 四、预测链 (Prediction Chain)

### 4.1 数据结构

```json
{
  "id": "P-20260703-001",
  "timestamp": "2026-07-03T14:00:00Z",
  "agent_id": "family-finance-expert",
  "prediction": "上证指数Q3突破4100",
  "confidence": 0.75,
  "verification_window": "2026-10-01",
  "verifiable_condition": "上证收盘价>4100 至少3个交易日",
  "previous_hash": "a1b2c3...",
  "hash": "d4e5f6..."
}
```

### 4.2 不可篡改性

- 每条记录含 `previous_hash` 链接前一条
- 修改任何一条 → 后续所有 hash 失效
- 验证结果不覆写，追加新记录
- 整条链可公开审计

### 4.3 完整性验证

```bash
python3 prediction_ledger.py status
# → 预测链: N 条记录, 完整性: OK/BROKEN
```

## 五、信任分 (Trust Score)

### 5.1 公式

```
Trust = accuracy × 0.45 + recency × 0.25 + volume × 0.15 + immune_bonus × 0.15

accuracy:  验证通过率(0-1)
recency:   最近验证加权(e^(-days/90))
volume:    预测总量对数(log10(total)/log10(1000))
immune:    免疫等级加成(L0=0, L1=0.05, L2=0.10, L3=0.15, L4=0.20, L5=0.25)
```

### 5.2 设计原则

- **accuracy 权重最高(45%)**：做对最重要
- **recency(25%)**：老黄历不值钱，半衰期 62 天
- **volume 对数缩放(15%)**：防刷量，1000 条才满分
- **immune 防作弊(15%)**：高免疫等级 Agent 更难被操纵

### 5.3 排行榜

```bash
python3 trust_score.py board

# Rank  Agent                    Trust   Acc   Verified
# 1     family-finance-expert    0.654   62%   156
# 2     family-research-expert   0.520   55%   89
# ...
```

## 六、路由优先原则

```
查询进入 → 匹配候选专家(关键词)
         → 按信任分排序
         → 最高分 = 主责专家
         → 难度>=4 → 增加次高分互补专家
```

类比比特币"最长链原则"——**不是最早注册的优先，是信任最高的优先。**

## 七、防御机制

### 7.1 女巫攻击防御

攻击：创建 1000 个假 Agent，互相验证虚假预测
防御：
- 免疫等级(L4+ 需要真实系统运行，无法伪造)
- 交叉验证要求(验证方也需要有信任分)
- 预测窗口(不能即时自验证)

### 7.2 刷量防御

攻击：大量低质量预测刷 volume
防御：
- volume 对数缩放(log10(1000) = 3, log10(100) = 2，边际递减)
- accuracy 权重最高(45%)，刷错误的量反而拉低总分

### 7.3 历史篡改防御

攻击：修改已完成的预测记录
防御：
- 链式哈希(修改一条 → 全链断裂)
- 公开注册表(GitHub Pages，Git 历史不可改)
